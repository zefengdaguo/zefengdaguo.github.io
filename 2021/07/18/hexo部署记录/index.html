本以为一个静态博客的搭建应该没什么难度，没想到还是耗费了不少时间，有必要做个记录。

安装Hexo

首先给 npm 设置代理，提升下载速度：

安装hexo

查看是否安装成功

有可能会提示找不到hexo命令，手动添加环境变量，everything搜索hexo-cli，在path中添加 …\hexo-cli\bin

创建博客项目

新建一个文件夹，在该目录下打开 git bash。

初始化项目

部署到github pages

免费的github pages只支持公共仓库。

创建一个public的github仓库，名为<username>.github.io。

编辑博客项目目录中的_config.yml

安装 hexo-deployer-git

部署

在github仓库settings/pages开启后，可以通过 zefengdaguo.github.io 访问博客。

自定义域名

现有域名 uneila.top，添加解析。

以阿里云DNS解析为例，为uneila.top添加一条解析记录。主机记录 @ 表示解析后的访问地址为 uneila.top

在github仓库的settings/pages中设置custom domain为uneila.top，建议勾选Enforce HTTPS 。

现在访问zefengdaguo.github.io后地址为uneila.top

在github上自定义域名后会在仓库中生成一个CNAME文件，内容为自定义的域名uneila.top。但是本地还没有设置，一旦hexo d又要再次在pages中操作。避免麻烦，在本地hexo项目的source目录中添加CNAME文件，这样hexo g后会在public目录中生成CNAME，重新部署后不必再次在网页上设置域名。

域名解析

以上部署了一个github pages的个人站点。接下来再创建一个仓库，熟悉一下github pages和域名解析。

非个人站点的仓库开启pages后的url为<username>.github.io/<repository>

在域名解析中新增一条子域名，主机记录为subblog

github创建subblog仓库；本地新建一个hexo项目，_config.yml如下

在source目录中添加CNAME文件，内容为 subblog.uneila.top ；

hexo g生成；hexo d部署

在subblog仓库的settings/pages中勾选Enforce HTTPS 。

现在访问zefengdaguo.github.io/subblog后地址为subblog.uneila.top
